CSI Data Quality Analyzer

Assessing 3D readiness of municipal building data for Urban Digital Twin applications

This project evaluates the readiness of the CSI building dataset of Turin for 3D city modelling and Urban Digital Twin use. It combines GIS-based data quality assessment, Python analysis, 2D and 3D web visualization, and a FIWARE-inspired transformation framework to move from fragmented municipal data toward structured UDT-ready entities.

Introduction & State of Art

Urban Digital Twins require datasets that are not only geometrically available, but also semantically structured, interoperable, and scalable. In practice, many municipal datasets were produced for conventional GIS or administrative purposes and do not directly match these requirements.

This project positions itself in that gap. It starts from a real authoritative dataset and asks whether the transition from GIS data to an Urban Digital Twin is actually feasible, and under which conditions.

Methodology

The methodology combines dataset inspection, geometry validation, semantic restructuring, and FIWARE-inspired modeling. It was designed not only to assess data quality, but also to test how a traditional GIS dataset can be transformed into a more interoperable Urban Digital Twin representation.

Geometry Analysis

Validation of polygon quality, height availability, and 3D extrusion potential across the study area.

Semantic Structuring

Integration of attributes from multiple layers to build a clearer object-level representation.

FIWARE Transformation

Reorganization of fragmented GIS features into entity-based structures with explicit properties and quality metadata.

Case Study: Background and Overview

The broader analysis focuses on CSI data from Circoscrizione 7 in Turin. Within this context, the Lavazza building was selected as a detailed case study because it allows object-level testing of geometry, Level of Detail, semantic consistency, and interoperability.

The case study makes it possible to move beyond city-scale metrics and examine how the methodology behaves when applied to one real building represented through multiple layers and polygons.

Model Validation and Application Results to the Case Study

4.1 Data Processing and Model Setup

Data preparation included spatial selection of the Lavazza building, geometry aggregation, extraction of semantic attributes, reprojection for web interoperability, and preparation of a clean building entity for visualization and FIWARE-inspired transformation.

4.2 Model Validation and Performance Testing

Validation was carried out by checking geometry coherence, height consistency, building visualization in 2D and 3D, and the internal consistency of the resulting building entity. Control parameters included geometry aggregation quality, height usability, and semantic plausibility.

4.3 Analysis of Results and Problem Solving

  • Missing or zero height values were filtered and interpreted through derived indicators.
  • Fragmented geometries were merged into a single logical building object.
  • Roof-related geometries were tested and identified as insufficient for real 3D roof reconstruction.
  • Mixed semantics were resolved through controlled attribute selection and restructuring.

4.4 Optimization and Synthesis of Findings

After identifying the main limitations, the dataset was reorganized into a cleaner, entity-based structure. This did not eliminate the original data limitations, but it improved clarity, interpretability, and readiness for future UDT integration.

FIWARE-Based UDT Transformation Framework (Proposed)

This project does not only describe FIWARE concepts. It proposes a practical transformation workflow to convert fragmented GIS data into a more structured Urban Digital Twin representation.

Proposed Workflow

Step 1 — Geometry Normalization

Fragmented geometries are aggregated into one coherent building representation and reprojected into EPSG:4326 for interoperable web use.

Step 2 — Semantic Structuring

Attributes from multiple layers are reorganized into a unified schema including height, floors, use, and construction metadata.

Step 3 — Relationship Definition

Implicit GIS relations are made explicit through entity-based logic such as hasPart and roof reference relationships.

Step 4 — Data Quality Encoding

LoD, geometry quality, semantic quality, and confidence indicators are encoded as metadata instead of remaining hidden.

Step 5 — FIWARE Entity Generation

The final output is a structured entity compatible with NGSI-style logic and better suited for UDT environments.

Discussion & Conclusion

The project shows that the main barrier to Urban Digital Twin implementation is not visualization alone, but data readiness. A dataset may support basic 3D representation while still lacking the semantic coherence and interoperability required for a more advanced digital twin environment.

By combining dataset-scale analysis with object-level restructuring, this work shows that FIWARE is useful not because it magically solves data limitations, but because it forces those limitations to become explicit, structured, and manageable.

Built as part of a master’s thesis on urban data quality, 3D city modelling, and Urban Digital Twin readiness.
